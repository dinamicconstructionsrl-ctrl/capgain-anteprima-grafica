CapGain — Anteprima nuova versione · come pubblicarla su Vercel
================================================================

Questi file sono un sito STATICO (solo HTML). Su Vercel NON serve build.
La home è index.html. Crea un PROGETTO NUOVO: così quello in produzione resta intatto.

------------------------------------------------------------
OPZIONE A — Vercel CLI (più veloce, senza Git)
------------------------------------------------------------
1. Scompatta questo zip in una cartella.
2. Installa la CLI:        npm i -g vercel
3. Entra nella cartella:   cd capgain-anteprima-vercel
4. Lancia:                 vercel
   - fai login se richiesto
   - "Set up and deploy?" → Yes
   - "Link to existing project?" → No  (vogliamo un progetto NUOVO)
   - Project name → es. capgain-anteprima
   - Framework preset → Other
   - lascia le altre risposte di default
   → ottieni un URL di preview (xxxxx.vercel.app)
5. Per l'URL definitivo:    vercel --prod

------------------------------------------------------------
OPZIONE B — Dashboard Vercel (con Git)
------------------------------------------------------------
1. Carica questa cartella in una nuova repo GitHub (o in una sottocartella).
2. Vercel → Add New → Project → importa la repo.
3. Framework Preset: Other · Build Command: (vuoto) · Output Directory: (vuoto/.)
4. Deploy. La home sarà index.html.

------------------------------------------------------------
Collegare il sottodominio anteprima.capgain.eu
------------------------------------------------------------
1. Nel progetto Vercel → Settings → Domains → Add → anteprima.capgain.eu
2. Vercel ti indica il record DNS da impostare (di solito un CNAME).
   Se il DNS di capgain.eu è già su Vercel/Cloudflare, è quasi un clic.
3. Fatto: la produzione (web.capgain.eu) resta separata e intatta.

Nota: le anteprime usano dati di esempio. Caratteri e icone si caricano da CDN
(Google Fonts, Tabler) quindi serve connessione internet quando le apri.
